    17→    17→## How to (re)start the dev server — IMPORTANT for cron runs
    18→    18→
    19→    19→This Kata container **kills background processes when the bash session ends**. `nohup`/`setsid`/`disown` alone are NOT enough because the `bun run dev` script pipes through `tee` (which breaks on session end). The reliable pattern that survives across bash calls is a **double-fork daemon running `next dev` directly** (no `tee` pipe), reparenting to init (PID 1 / tini):
    20→    20→
    21→    21→```bash
    22→    22→cd /home/z/my-project
    23→    23→pkill -f "next dev" 2>/dev/null; sleep 1
    24→    24→( setsid bash -c 'exec bunx next dev -p 3000' </dev/null >>/home/z/my-project/dev.log 2>&1 & )
    25→    25→sleep 12
    26→    26→curl -s -m 15 -w "HTTP %{http_code}\n" http://localhost:3000/ -o /dev/null
    27→    27→```
    28→    28→
    29→    29→Every webDevReview cron run should execute this at the start so the server is alive for QA.
